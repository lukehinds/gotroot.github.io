1 Deployment of Contrail Roles

1.1 Deploy quickstart

Cherry pick Carlos patch (unless its already merged) with a refspec --extra-var

You likely won't need the overcloud_image_url. I had this as the master image had not yet been pushed to the CDN. 

./quickstart.sh \
  --working-dir $WS \
  --teardown all \
  --tags all \
  --skip-tags overcloud-validate,overcloud-deploy \
  --config ./config/general_config/minimal.yml \
  --release master \
  --extra-vars undercloud_image_url=https://images.rdoproject.org/master/delorean/consistent/stable/undercloud.qcow2 \
  --extra-vars overcloud_templates_refspec=refs/changes/96/368196/2 \
  $VIRTHOST

1.2 Append the Contrail Role

Update tripleo-heat-templates/roles_data.yaml by appending the following: 

- name: Contrail
  CountDefault: 1
  ServicesDefault:
    - OS::TripleO::Services::CACerts
    - OS::TripleO::Services::ContrailAnalytics
    - OS::TripleO::Services::ContrailConfig
    - OS::TripleO::Services::ContrailControl
    - OS::TripleO::Services::ContrailDatabase
    - OS::TripleO::Services::ContrailWebui
    - OS::TripleO::Services::Kernel
    - OS::TripleO::Services::Ntp
    - OS::TripleO::Services::Timezone
    - OS::TripleO::Services::TripleoPackages
    - OS::TripleO::Services::TripleoFirewall
    - OS::TripleO::Services::SensuClient
    - OS::TripleO::Services::FluentdClient
    - OS::TripleO::Services::VipHosts

1.3 Deploy Overcloud referencing the contrail templates

source stackrc
openstack overcloud image upload
openstack baremetal import instackenv.json
openstack baremetal introspection bulk start
neutron subnet-update <id> --dns-nameserver 8.8.8.8  
openstack overcloud deploy \ 
  --libvirt-type qemu \ 
  --ntp-server pool.ntp.org \
  --templates /home/stack/tripleo-heat-templates
