FIPS 140-2 Kernel in RHEL / CentOS 7 Guest Images

The following steps will enable a FIPS-140-2 kernel in RHEL / CentOS 7. 

yum install dracut-fips

Check the CPU supports AES-NI 

grep -qw aes /proc/cpuinfo && echo YES || echo no

If a Yes is returned, install dracut-fips-aesni 

yum install dracut-fips-aesni

Check if prelink is enabled 

rpm -q prelink && sed -i '/^PRELINKING/s,yes,no,' /etc/sysconfig/prelink

If it is enabled, remove it: 

rpm -q prelink && prelink -uav

Backup existing initramfs 

mv -v /boot/initramfs-$(uname -r).img{,.bak}

Run dracut 

dracut

Update GRUB with fips=1 

grubby --update-kernel=$(grubby --default-kernel) --args=fips=1

If you have a seperate /boot partition to rootfs, then run the following: 

uuid=$(findmnt -no uuid /boot)
[[ -n $uuid ]] && grubby --update-kernel=$(grubby --default-kernel) --args=boot=UUID=${uuid}

Reboot the system and use sysctl to check the Kernel is running in FIPS mode: 

sysctl crypto.fips_enabled

This should return 1 if everything went accordingly.
